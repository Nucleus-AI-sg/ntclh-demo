# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A frontend-only interactive prototype of **Nucleus AI**, an end-to-end career placement programme management platform proposed for NTUC LearningHub (NTUC LHub). Uses mock data and simulated AI responses - no backend, no database, no live API calls.

## Tech Stack

- **Framework**: Next.js (App Router) with TypeScript
- **Styling**: Tailwind CSS + shadcn/ui
- **Charts**: Recharts
- **Data**: Static JSON fixtures (no backend)
- **AI simulation**: Pre-scripted responses with typing animation
- **Deployment**: Vercel (static export)

## Development Commands

```bash
npm run dev          # Start dev server (localhost:3000)
npm run build        # Production build (static export)
npm run lint         # ESLint
npm run type-check   # TypeScript compiler check (tsc --noEmit)
```

## Architecture

### Routing (App Router)

Each page maps to a route under `src/app/`. Dynamic routes use `[id]` folders.

| Route | PRD | Page |
|-------|-----|------|
| `/` | PRD-02 | Coordinator Dashboard |
| `/enrolment` | PRD-03 | Enrolment & AI Assessment |
| `/tracking` | PRD-04 | Trainee Tracking & Verification |
| `/matching` | PRD-05 | Placement Matching |
| `/analytics` | PRD-06 | Analytics & Reporting |
| `/assistant` | PRD-07 | AI Chat Assistant |
| `/trainee/[id]` | PRD-08 | Trainee Profile (360-degree view) |
| `/employers` | PRD-09 | Employer List |
| `/employer/[id]` | PRD-09 | Employer Profile |
| `/programmes` | PRD-10 | Programme List |
| `/programme/[id]` | PRD-10 | Programme Detail |
| `/communications` | PRD-11 | Communications Hub |
| `/documents` | PRD-12 | Document Centre |
| `/ai-insights` | PRD-13 | AI Insights & Model Performance |
| `/settings` | PRD-14 | Integrations & Settings |

**Always read the relevant PRD before building or modifying a page.**

### Data Layer

All mock data lives in `src/data/` as static JSON/TS fixtures. Pages import data directly - no API calls, no fetch, no server actions. All entities share a consistent mock universe (see `prds/PRD-00-demo-overview.md` for full details).

### Shared Layout

- **Sidebar navigation** (persistent left sidebar):
  - Operations: Dashboard, Enrolment, Tracking, Matching
  - Management: Programmes, Employers, Communications, Documents
  - Intelligence: Analytics, AI Insights, Assistant
  - System: Integrations & Settings
- **Top bar**: Nucleus AI logo, user name ("Sarah Tan, Programme Coordinator"), notification bell
- **Breadcrumbs**: Page context trail below the top bar
- **Cross-linking**: Trainee names link to `/trainee/[id]`, employer names to `/employer/[id]`, programme names to `/programme/[id]`
- **Responsive**: Desktop-first, functional on tablet

### Design Reference

The `stitch-html/` folder contains static HTML prototypes for each page. Use these as visual/layout reference when building components. They are not functional code. The numbering maps to PRDs (e.g. `01-coordinator-dashboard.html` corresponds to PRD-02).

## Coding Conventions

- British English throughout (colour, behaviour, organised)
- Never use em dashes - use hyphens, commas, or separate sentences
- Keep code files under 200 lines (excluding test and data files)
- Favour composition over inheritance
- Use shadcn/ui components wherever possible
- Colour-coded status badges: consistent colour mapping across all pages (green = verified/high, amber = pending/medium, red = flagged/low)
- All AI scores display as percentages with colour coding (green >80%, amber 60-80%, red <60%)

## Constraints

- No backend, database, or external API calls
- No real SingPass/Myinfo integration (simulated form pre-fill)
- No real OCR processing (simulated verification results)
- AI chat responses are pre-scripted, not generated by a live LLM
- All trainee data is fictional
- Document viewer uses embedded mock PDFs/images
